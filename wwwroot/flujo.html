Flujo del proyecto PDF a TXT

Esta pagina explica, de forma sencilla, que hace cada parte del proyecto cuando se suben archivos PDF.

Diagrama general

Usuario sube PDFs
v
Backend recibe archivos
v
Se extrae texto
v
Se buscan citas y referencias APA 7
v
La web muestra resultados

Flujo paso a paso

1

La pagina recibe los PDFs

El usuario selecciona archivos o arrastra una carpeta. La pagina junta los PDFs y los manda al backend.

v
2

Program.cs coordina la solicitud

Revisa que si sean PDFs, prepara nombres seguros para los TXT y llama a los servicios que hacen el trabajo pesado.

v
3

PdfTextService intenta leer texto

Primero intenta sacar texto directo del PDF. Esto funciona cuando el documento tiene texto seleccionable.

v
4

Si no hay texto seleccionable, informa el problema

Cuando no hay texto directo, el sistema no intenta convertir imagenes. Responde que no se pudo encontrar texto seleccionable en el documento.

v
5

Apa7ReferenceService busca referencias

Esta parte revisa el texto completo y primero localiza una seccion llamada referencias, referencias bibliograficas, bibliografia o references. Desde ahi toma el bloque final del documento y lo analiza como lista de fuentes.

Despues separa cada referencia usando el formato comun de APA 7: autor, fecha entre parentesis, titulo y fuente. Por eso reconoce autores personales como Ruiz-Espinoza, F. H. y tambien instituciones como Subsecretaria de Educacion Media Superior.

El objetivo no es entender todo el contenido del articulo, sino identificar donde empieza cada referencia para mostrarla como una coincidencia independiente.

Tambien busca citas parenteticas fuera de referencias

Antes de analizar la bibliografia, el sistema toma solo el cuerpo del documento: todo lo que aparece antes de la seccion de referencias, referencias bibliograficas, bibliografia o references. En esa parte busca citas entre parentesis como (Ramirez, 2021), (Chavez R., 2017) o (Ramirez & Lopez, 2021).

Cada PDF regresa su propia lista de citas parenteticas encontradas y cuantas veces aparece cada una. Las citas ubicadas dentro de la bibliografia no se cuentan.

Pseudocodigo: citas parenteticas

Snippet: buscar citas parenteticas

public static IReadOnlyList<ParentheticalCitation> FindParentheticalCitations(string text)
{
    var bodyText = ExtractTextBeforeReferenceSection(text);

    if (bodyText.Length == 0)
    {
        return [];
    }

    var matches = Regex.Matches(
            bodyText,
            @"\((?<citation>[^()]{2,220}(?:19|20)\d{2}[a-z]?[^()]*)\)")
        .Cast<Match>()
        .Select(match => TextUtilities.NormalizeWhitespace($"({match.Groups["citation"].Value})"))
        .Where(IsLikelyParentheticalCitation)
        .ToList();

    return matches
        .GroupBy(citation => citation, StringComparer.CurrentCultureIgnoreCase)
        .Select(group => new ParentheticalCitation(group.First(), group.Count()))
        .OrderBy(citation => citation.Citation, StringComparer.CurrentCultureIgnoreCase)
        .ToList();
}

Explicacion tecnica: esta parte trabaja con texto plano. Primero reduce el problema a una sola seccion del documento y luego usa expresiones regulares para detectar donde inicia cada referencia APA 7. Tambien limpia encabezados como REFERENCIAS, REFERENCIAS BIBLIOGRAFICAS o #1REFERENCIAS BIBLIOGRAFICAS cuando vienen pegados a la primera referencia.

public static List<string> FindReferences(string text)
{
    var section = ExtractReferenceSection(text);

    if (section.Length == 0)
    {
        return [];
    }

    var references = SplitReferences(section);
    return references.Count > 0 ? references : [section];
}

Separacion en texto corrido

Si el texto llega sin saltos de linea, el sistema busca cada aparicion de Apellido, Inicial. (año) o Nombre institucional. (año). Cuando encuentra otro inicio valido, corta la referencia anterior antes de analizarla. Si una referencia termina en DOI o URL, la siguiente puede empezar inmediatamente despues. 

funcion separarReferencias(seccion):

Como leer esos regex

Un regex es una regla para buscar texto. En el codigo real se construye un patron grande uniendo tres patrones pequenos: fecha, autor personal y autor institucional.

1. Fecha APA 7

const string datePattern =
    @"\(\s*(?:n\.?\s*d\.?|s\.?\s*f\.?|(?:19|20)\d{2}[a-z]?)(?:,\s*[^)]{1,40})?\s*\)";

Este patron busca la fecha que va despues del autor. Se lee asi: 

\(
    busca el parentesis de apertura "("

\s*
    permite espacios despues del parentesis

(?:n\.?\s*d\.?|s\.?\s*f\.?|(?:19|20)\d{2}[a-z]?)
    acepta una de estas opciones:
    - n.d.  -> sin fecha en ingles
    - s.f.  -> sin fecha en español
    - 19xx o 20xx, por ejemplo 1997, 2021, 2021a

(?:,\s*[^)]{1,40})?
    permite texto extra dentro del parentesis, por ejemplo fechas mas largas

\s*\)
    permite espacios y busca el parentesis de cierre ")"

2. Autor personal

const string personAuthorPattern =
    @"[A-ZÁÉÍÓÚÑ][A-Za-zÁÉÍÓÚÜÑáéíóúüñ'\-]+,\s*(?:[A-ZÁÉÍÓÚÜÑ][a-záéíóúüñ]{0,2}\.?\s*){1,4}(?:,\s*&\s*|,\s*|;\s*&?\s*|&\s*)?";

Este patron busca autores con forma APA 7 de persona. Por ejemplo: Ruiz-Espinoza, F. H. o Varela, M. 

[A-ZÁÉÍÓÚÑ]
    exige que el apellido empiece con mayuscula

[A-Za-zÁÉÍÓÚÜÑáéíóúüñ'\-]+
    toma el resto del apellido
    permite letras, apostrofe y guion
    por eso acepta "Ruiz-Espinoza"

,\s*
    exige una coma despues del apellido
    permite espacios despues de la coma

(?:[A-ZÁÉÍÓÚÜÑ][a-záéíóúüñ]{0,2}\.?\s*){1,4}
    busca entre 1 y 4 iniciales
    acepta "M.", "F. H.", "Ma. J."

(?:,\s*&\s*|,\s*|;\s*&?\s*|&\s*)?
    permite separadores entre autores
    por ejemplo "," o "&"

3. Autor institucional

const string groupAuthorPattern =
    @"[A-ZÁÉÍÓÚÑ][A-Za-zÁÉÍÓÚÜÑáéíóúüñ0-9&'\-]*(?:\s+(?:de|del|la|las|los|el|en|y|e|of|the|and|for|[A-ZÁÉÍÓÚÑ][A-Za-zÁÉÍÓÚÜÑáéíóúüñ0-9&'\-]*)){1,14}\.\s*(?:\([^)]{2,40}\)\.\s*)?";

Este patron busca instituciones como autor. Por ejemplo: Instituto Cervantes., Cervantes., Comision Nacional de Bioetica. o Subsecretaria de Educacion Media Superior. (DGETAyCM). 

[A-ZÁÉÍÓÚÑ]
    exige que el nombre de la institucion empiece con mayuscula

[A-Za-zÁÉÍÓÚÜÑáéíóúüñ0-9&'\-]*
    toma el resto de la primera palabra

(?:\s+(?:de|del|la|las|los|el|en|y|e|of|the|and|for|PALABRA)){0,14}
    permite que el nombre tenga varias palabras
    tambien permite instituciones de una sola palabra, como "Cervantes."
    acepta conectores como "de", "del", "la", "y"
    acepta palabras con mayuscula como "Educacion", "Media", "Superior"
    permite de 1 a 14 palabras extra

\.\s*
    exige punto al final del nombre institucional

(?:\([^)]{2,40}\)\.\s*)?
    acepta una sigla opcional entre parentesis
    ejemplo: (DGETAyCM).

4. Como se unen los patrones

var startMatches = Regex.Matches(
    text,
    $@"(?<!\p{{L}})(?<entryStart>(?:{personAuthorPattern}(?:(?:{personAuthorPattern}){{0,19}})|{groupAuthorPattern}){datePattern})",
    RegexOptions.CultureInvariant);

Esta linea busca todos los lugares donde podria empezar una referencia. La regla completa dice: empieza con autor personal o autor institucional, y despues debe venir una fecha APA 7. 

(?:personAuthorPattern ... | groupAuthorPattern)
    acepta uno de dos caminos:
    - autor personal
    - autor institucional

{datePattern}
    despues del autor debe aparecer la fecha

Regex.Matches(...)
    regresa todas las posiciones donde encontro ese patron

Por eso una referencia institucional como Subsecretaria de Educacion Media Superior. (DGETAyCM). (2022). se interpreta asi: 

Subsecretaria de Educacion Media Superior.  -> autor institucional
(DGETAyCM).                                -> sigla opcional
(2022).                                    -> fecha APA 7
Guia para el diseño...                     -> titulo

Y una referencia personal como Ruiz-Espinoza, F. H., & Pineda-Castillo, K. A. (2021). se interpreta asi: 

Ruiz-Espinoza, F. H.     -> primer autor personal
&                         -> separador entre autores
Pineda-Castillo, K. A.    -> segundo autor personal
(2021).                   -> fecha APA 7

Se cuentan los años citados

Una vez separadas y analizadas las referencias, el sistema cuenta años solo en las referencias que cumplen APA 7. En esas referencias busca el primer año que aparece entre parentesis. Por ejemplo, en (2021) toma el año 2021. Si una referencia tiene letras como (2021a), tambien cuenta el año como 2021.

Luego suma cuantas referencias APA 7 correctas de cada documento pertenecen a cada año. Con esos conteos construye una tabla comparativa: las filas son años y las columnas son los archivos PDF.

Si se suben dos documentos, la tabla permite ver rapidamente en que años se concentran las citas de cada uno. Un valor 0 significa que ese documento no tuvo referencias detectadas para ese año.

Explicacion tecnica: el backend primero filtra IsApa7Compliant. Despues regresa un diccionario por archivo. La llave es el año y el valor es cuantas referencias APA 7 correctas de ese archivo tienen ese año.

Pseudocodigo

funcion contarAnios(referencias):
    conteo = diccionario vacio

    referenciasCorrectas = filtrar referencias donde IsApa7Compliant sea verdadero

    para cada referencia en referenciasCorrectas:
        anio = buscar primer año entre parentesis

        si encontro anio:
            conteo[anio] = conteo[anio] + 1

    regresar conteo ordenado por año

Como se ve en el codigo

public static SortedDictionary<int, int> CountYears(IEnumerable<string> references)
{
    var counts = new SortedDictionary<int, int>();

    foreach (var reference in references)
    {
        var match = Regex.Match(
            reference,
            @"\(\s*(?<year>(?:19|20)\d{2})[a-z]?\s*\)");

        if (!match.Success)
        {
            continue;
        }

        var year = int.Parse(match.Groups["year"].Value);
        counts[year] = counts.TryGetValue(year, out var current)
            ? current + 1
            : 1;
    }

    return counts;
}

Despues, el frontend recibe esos conteos y arma la tabla. Primero junta todos los años encontrados en todos los documentos, luego crea una fila por cada año.

Se cuentan los primeros autores

Ademas del año, el sistema identifica el primer autor solo en referencias que cumplen APA 7. Si la referencia empieza con autor personal, toma el apellido y las iniciales. Si empieza con institucion, toma el nombre institucional antes de la fecha.

Con ese dato construye un conteo por documento: cuantas referencias APA 7 correctas pertenecen a cada primer autor. Esto ayuda a comparar rapidamente que autores o instituciones aparecen con mas frecuencia en cada PDF.

Explicacion tecnica: el backend regresa un diccionario llamado authorCounts. La llave es el primer autor y el valor es cuantas referencias APA 7 correctas empiezan con ese autor.

            primerAutor = primer autor personal
        si no:
            primerAutor = autor institucional

Como se ve en el codigo

public static SortedDictionary<string, int> CountFirstAuthors(IEnumerable<string> references)
{
    var counts = new SortedDictionary<string, int>();

    foreach (var reference in references)
    {
        var firstAuthor = ExtractFirstAuthor(reference);

        if (firstAuthor.Length == 0)
        {
            continue;
        }

        counts[firstAuthor] = counts.TryGetValue(firstAuthor, out var currentCount)
            ? currentCount + 1
            : 1;
    }

    return counts;
}

Se clasifican y validan los casos de referencias

Despues de separar las referencias, el sistema revisa si cada una parece APA 7 o si pertenece a otro estilo. Cuando detecta APA 7, intenta clasificar el tipo de fuente y valida campos obligatorios segun ese tipo.

Estilos que detecta

  • APA 7: autor, fecha entre parentesis, titulo y fuente.
  • IEEE: numeracion con corchetes o inicial antes del apellido.
  • Vancouver: numeracion inicial y abreviaturas de revista medica.
  • MLA: autor con nombre completo y titulo entre comillas.
  • Chicago: autor, titulo, ciudad/editorial y año al final.
  • Otro estilo: cuando no coincide con las reglas anteriores.

Tipos APA 7 que clasifica

  • Articulo de revista: revisa volumen y DOI o URL. El numero de issue y las paginas son opcionales.
  • Capitulo de libro: revisa "En", editor, paginas con pp. y editorial.
  • Capitulo de libro de actas: igual que capitulo, pero reconoce proceedings, conferencia, congreso, actas o memorias.
  • Reporte institucional: revisa autor institucional completo, editorial institucional o URL.
  • Libro: revisa autor, año, titulo, editorial y formato de edicion.
  • Sitio web: revisa nombre del sitio y URL.
  • Tipo no estandar: por ejemplo referencias con entrevistador.
  • Ambiguo: cuando parece APA 7, pero no hay suficientes datos para ubicar el tipo exacto.

Reglas generales que valida

  • Que exista fecha despues del autor.
  • Que el autor sea personal en formato Apellido, iniciales o institucion.
  • Que multiples autores usen & antes del ultimo autor.
  • Que el DOI este como URL completa: https://doi.org/...
  • Que ORCID no se use como sustituto del DOI o URL de la fuente.
  • Que no use frases antiguas como Retrieved from, Recuperado de u Obtenido de.
  • Que el año no aparezca al final en vez de despues del autor.
  • Que referencias con mas de 20 autores usen puntos suspensivos antes del ultimo autor.
  • Que despues de la fecha se distingan titulo y fuente.

Snippet: detectar el estilo

Primero el codigo decide si la referencia parece IEEE, Vancouver, MLA, Chicago, APA 7 u otro estilo. Solo si detecta APA 7 pasa a la validacion detallada. 

private static string DetectCitationStyle(string reference)
{
    if (IsIeeeReference(reference))
    {
        return "Formato IEEE";
    }

    if (IsVancouverReference(reference))
    {
        return "Formato Vancouver";
    }

    if (IsMlaReference(reference))
    {
        return "Formato MLA";
    }

    if (IsChicagoReference(reference))
    {
        return "Formato Chicago";
    }

    if (LooksLikeApa7(reference))
    {
        return "APA 7";
    }

    return "Otro estilo";
}

Snippet: saber si parece APA 7

Para considerar una referencia como APA 7, debe tener una fecha entre parentesis y antes de esa fecha debe existir un autor personal o institucional con formato reconocible. 

private static bool LooksLikeApa7(string reference)
{
    var dateMatch = Regex.Match(reference, DatePattern, RegexOptions.CultureInvariant);

    if (!dateMatch.Success || dateMatch.Index <= 0)
    {
        return false;
    }

    var authorText = TextUtilities.NormalizeWhitespace(reference[..dateMatch.Index]);

    return Regex.IsMatch(authorText, $@"^(?:{PersonAuthorPattern})+", RegexOptions.CultureInvariant) ||
        Regex.IsMatch(authorText, $@"^{GroupAuthorPattern}$", RegexOptions.CultureInvariant);
}

Snippet: clasificar el tipo APA 7

Cuando ya parece APA 7, el sistema busca pistas: palabras de conferencia, uso de En, autor institucional, volumen y paginas, editorial o URL. Si detecta editorial sin nombre de revista, clasifica como libro aunque tambien tenga DOI. 

private static string ClassifyReference(string reference)
{
    var dateMatch = Regex.Match(reference, DatePattern, RegexOptions.CultureInvariant);
    var authorText = dateMatch.Success
        ? TextUtilities.NormalizeWhitespace(reference[..dateMatch.Index])
        : string.Empty;

    if (Regex.IsMatch(reference, @"\((?:[^)]*Entrevistador[^)]*)\)", RegexOptions.IgnoreCase))
    {
        return "Tipo no estandar";
    }

    if (IsProceedingsSource(reference))
    {
        return "Capitulo de libro de actas";
    }

    if (Regex.IsMatch(reference, @"\bEn\s+.+\(Eds?\.?\)", RegexOptions.CultureInvariant))
    {
        return "Capitulo de libro";
    }

    if (Regex.IsMatch(authorText, $@"^{GroupAuthorPattern}$", RegexOptions.CultureInvariant))
    {
        return "Reporte institucional";
    }

    if (HasBookPublisher(reference) && !LooksLikeAcademicSource(reference))
    {
        return "Libro";
    }

    if (HasJournalVolume(reference) ||
        Regex.IsMatch(reference, @"\b\d+\s*\(\s*\d+\s*\)\s*,\s*\d+", RegexOptions.CultureInvariant))
    {
        return "Articulo de revista";
    }

    if (HasBookPublisher(reference))
    {
        return "Libro";
    }

    if (Regex.IsMatch(reference, @"https?://\S+", RegexOptions.CultureInvariant))
    {
        return LooksLikeAcademicSource(reference) ? "Articulo de revista" : "Sitio web";
    }

    return "Ambiguo";
}

Snippet: validar campos por tipo

Cada tipo tiene reglas propias. Por ejemplo, un articulo de revista necesita volumen y DOI o URL. El numero de issue puede aparecer, pero no se marca como error si falta. Tampoco se marca error si faltan paginas, porque algunas revistas usan numero de articulo o identificador DOI en lugar de paginacion tradicional. Un capitulo necesita En, editor, paginas y editorial. Un libro con editorial y DOI sigue siendo libro, no articulo de revista. 

private static IEnumerable<string> ValidateByType(string reference, string referenceType)
{
    var reasons = new List<string>();

    switch (referenceType)
    {
        case "Articulo de revista":
            if (!HasJournalVolume(reference))
            {
                reasons.Add("No cumple: articulo de revista sin volumen detectable.");
            }

            if (!Regex.IsMatch(reference, @"https?://\S+"))
            {
                reasons.Add("No cumple: articulo de revista sin DOI o URL.");
            }
            break;

        case "Capitulo de libro":
        case "Capitulo de libro de actas":
            if (!Regex.IsMatch(reference, @"\bEn\s+"))
            {
                reasons.Add("No cumple: capitulo de libro/actas sin 'En'.");
            }

            if (!Regex.IsMatch(reference, @"\b\(Eds?\.?\)|\b\(Ed\.?\)"))
            {
                reasons.Add("No cumple: capitulo de libro/actas sin editor.");
            }

            if (!Regex.IsMatch(reference, @"\(pp\.\s*\d+\s*[-–]\s*\d+\)"))
            {
                reasons.Add("No cumple: capitulo de libro/actas sin paginas.");
            }
            break;
    }

    return reasons;
}

Snippet: decidir si cumple

Al final, el sistema separa las razones que empiezan con No cumple:. Si no hay razones de error, la referencia se marca como correcta. Si hay al menos una, se marca como APA 7 con errores. No modifica ni reescribe la referencia original. 

var blockingReasons = reasons
    .Where(reason => reason.StartsWith("No cumple:", StringComparison.Ordinal))
    .ToList();

if (blockingReasons.Count == 0)
{
    reasons.Insert(0, "Cumple estructura APA 7 basica para el tipo detectado.");
}

return new ReferenceAnalysis(
    number,
    normalized,
    referenceType,
    citationStyle,
    blockingReasons.Count == 0 ? "Apa7Correct" : "Apa7WithErrors",
    blockingReasons.Count == 0,
    reasons);

El sistema no corrige referencias. Es un validador escolar basado en patrones: marca casos probables y explica las razones para que el usuario revise la referencia.

La web muestra el resultado

El frontend muestra cada archivo, las referencias detectadas, si cumplen o no APA 7, las razones del analisis, el conteo de años citados y el conteo de primeros autores. Las tablas de conteo incluyen totales por fila, por archivo y un total general.

Tambien permite exportar un reporte HTML con los resultados finales. Ese reporte incluye el resumen por archivo, las citas parenteticas, las referencias analizadas en APA 7 y las tablas de conteo.

Que hace cada archivo principal

Program.cs

Es la entrada del backend. Recibe los PDFs, llama a los servicios y responde a la pagina web.

PdfTextService.cs

Extrae texto seleccionable con PdfPig y cuenta los caracteres utiles.

Apa7ReferenceService.cs

Encuentra citas parenteticas, encuentra referencias, detecta estilos, clasifica tipos APA 7, valida reglas, cuenta años citados y cuenta primeros autores solo en referencias APA 7 correctas.

wwwroot/app.js

Controla la pagina: sube archivos, recibe resultados y dibuja las listas desplegables.

wwwroot/styles.css

Define la apariencia visual de la pagina principal.
