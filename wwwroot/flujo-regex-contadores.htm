Diagrama de flujo de regex y conteos APA 7

Esta pagina explica el codigo como una clase basica de programacion: primero vemos que entra, luego que preguntas hace el programa, y finalmente que datos cuenta. No incluye endpoints, carga de archivos ni descarga.

Idea principal: el programa no "entiende" una referencia como una persona. Lo que hace es buscar patrones de texto. Esos patrones se llaman regex. Si encuentra una forma parecida a Apellido, I. (2020)., la trata como posible APA 7.
Entrada Texto extraido del PDF con posible ruido de encabezado, pie, saltos y bloques mezclados.
Decision tecnica Regex para localizar seccion, cortar referencias APA/IEEE y clasificar formato base.
Salida Conteos estrictos solo desde referencias con IsApa7Compliant.
Proceso normal Decision / validacion Salida valida Salida de error o manual

0. Antes del diagrama: conceptos basicos

clase rapida
1

Regex

Una regex es una regla para buscar texto. Por ejemplo, puede buscar un año escrito como (2020).

  • No lee significado.
  • Busca formas repetidas.
  • Sirve para detectar autores, años, DOI y fuentes.
2

Separar

Antes de contar, el programa necesita cortar el texto en referencias individuales.

  • Busca donde empieza cada referencia.
  • Corta desde ese inicio hasta el siguiente inicio.
  • Si hay mezcla APA e IEEE, corta cada una por separado.
3

Contar

El conteo ocurre al final, cuando ya se sabe cuales referencias cumplen APA 7.

  • No cuenta referencias con errores.
  • No cuenta IEEE como APA.
  • Cuenta años, titulos, autores y editorial/revista.

1. Separacion tecnica de referencias

regex + limpieza + delimitacion

Esta parte responde una pregunta basica: "donde empieza y termina cada referencia?". El programa limpia texto que no pertenece a la bibliografia y luego busca inicios repetidos.

Como leer el diagrama: las cajas azules son acciones, los rombos amarillos son preguntas, las cajas verdes son resultados utiles y las rojas son casos que deben revisarse o no se pueden usar como APA 7.
1. NormalizarDejar el texto en una forma mas facil de leer para la computadora.
2. Localizar iniciosBuscar frases que parecen inicio de referencia, como autor + año.
3. CortarTomar el texto entre un inicio y el siguiente inicio.
4. AdvertirSi algo no es claro, avisar en lugar de inventar una respuesta.
El diagrama puede desplazarse horizontalmente en pantallas pequenas.

2. Clasificacion y conteos

APA 7 estricta + agregaciones

Esta parte responde otra pregunta: "esta referencia se puede contar como APA 7 correcta?". Primero se identifica el formato. Solo despues se revisan los campos obligatorios.

Punto clave: clasificar no es lo mismo que contar. Una referencia puede existir, pero si es IEEE o tiene errores, no entra al conteo estricto de APA 7.
1. Formato basePrimero pregunta: "parece APA 7 o parece otro estilo?".
2. Tipo APASi es APA, pregunta si es articulo, libro, capitulo, etc.
3. VeredictoDecide si cumple, tiene errores, no cumple o es otro formato.
4. ConteoSolo cuenta las referencias que pasaron todas las reglas.
Las flechas verdes terminan en datos que si se agregan a los conteos estrictos.

2.1 Que se cuenta y que no se cuenta

regla para principiantes

Para evitar confusiones, piensa en el conteo como una puerta final. Solo pasan las referencias que ya fueron separadas, clasificadas y aprobadas como APA 7.

Caso Que hace el programa Entra al conteo?
Referencia APA 7 correcta Extrae año, titulo, primer autor y editorial/revista. Si.
APA 7 con errores Muestra los errores, pero no la usa para los conteos estrictos. No.
IEEE, Vancouver, MLA o Chicago La marca como otro formato y no intenta corregirla como APA. No.
Referencia no delimitada La manda a revision manual porque el corte no es seguro. No.

3. Revision del proceso contra el codigo

lectura tecnica
  • Primero se prepara el texto: el codigo busca la seccion de referencias, limpia ruido y normaliza espacios antes de analizar.
  • Despues se hacen cortes: APA e IEEE se detectan por inicios distintos; luego se ordenan esos inicios para separar referencias individuales.
  • Despues se clasifica: IEEE, Vancouver, MLA y Chicago no se revisan como APA; se guardan como otro formato.
  • Despues se valida: si parece APA 7, se revisan autor, año, titulo, fuente, DOI/URL y reglas del tipo de documento.
  • Al final se cuenta: anio, titulo, primer autor y editorial/revista se calculan solo despues de filtrar con IsApa7Compliant.
  • Detalle importante: editorial/revista se extrae desde ReferenceAnalysis, porque depende de si la referencia es articulo, libro, sitio web, reporte, etc.

private const string DatePattern =
    @"\(\s*(?:n\.?\s*d\.?|s\.?\s*f\.?|(?:\d{1,2}\s+de\s+[A-Za-zÁÉÍÓÚÜÑáéíóúüñ]+\s+de\s+)?(?:19|20)\d{2}[a-z]?)(?:,\s*[^)]{1,40})?\s*\)";

private const string PersonAuthorPattern =
    SurnamePattern + @",\s*(?:" + InitialsPattern + @"){1,4}(?:,\s*&\s*|,\s*|;\s*&?\s*|&\s*|,\s*y\s*|\s+y\s+)?";

private const string GroupAuthorPattern =
    @"[A-Za-zÁÉÍÓÚÜÑÇĞİÖŞáéíóúüñçğıöş0-9&'\-]{2,}(?:\s+(?:de|del|la|las|los|el|en|y|e|of|the|and|for|[A-Za-zÁÉÍÓÚÜÑÇĞİÖŞáéíóúüñçğıöş0-9&'\-]{2,})){0,14}\.\s*(?:\([^)]{2,40}\)\.\s*)?";

private const string IeeeReferenceStartPattern =
    SurnamePattern + @",\s*(?:" + InitialsPattern + @"){1,4}(?:,\s+|:\s+)";
DatePattern: busca la fecha de la referencia. Acepta años como (2020), fechas en español como (15 de enero de 2020) y casos sin fecha como (s.f.) o (n.d.).
PersonAuthorPattern: busca autores personales con estructura Apellido, I.. Tambien permite varios autores separados por coma, punto y coma, & o y.
GroupAuthorPattern: busca autores institucionales, por ejemplo Organizacion Mundial de la Salud.. Se usa cuando el autor no es una persona.
IeeeReferenceStartPattern: busca referencias que empiezan parecido a IEEE: Apellido, I., o Apellido, I.:. Sirve para separarlas sin tratarlas como APA 7.
Apa7ReferenceService.cs - entrada del flujo 
public static List<string> FindReferences(string text)
{
    var section = ExtractReferenceSection(text);

    if (section.Length == 0)
    {
        return [];
    }

    var references = SplitReferences(section);
    return references.Count > 0 ? references : [section];
}
Linea 1: FindReferences recibe todo el texto del PDF en la variable text.
Busca la seccion: ExtractReferenceSection(text) intenta quedarse solo con la parte de referencias o bibliografia.
Si no hay seccion: cuando section.Length == 0, devuelve [], que significa lista vacia.
Si si hay seccion: SplitReferences(section) corta el bloque en referencias individuales.
Salida: si pudo separar, regresa la lista separada; si no pudo, regresa el bloque completo para no perder informacion.
Apa7ReferenceService.cs - separacion individual 
private static List<string> SplitReferences(string section)
{
    var text = RemoveIeeeInterReferencePageNumbers(
        TextUtilities.NormalizeWhitespace(RemoveReferenceSectionTitle(section)));

    var starts = FindAllReferenceStartIndexes(text).ToList();

    var references = new List<string>();

    for (var index = 0; index < starts.Count; index++)
    {
        var start = starts[index];
        var end = index + 1 < starts.Count ? starts[index + 1] : text.Length;
        var reference = CleanReference(text[start..end]);

        if (reference.Length > 0)
        {
            references.Add(reference);
        }
    }

    return references;
}
Normaliza: primero limpia el titulo de la seccion y espacios raros para que los regex trabajen mejor.
Busca inicios: FindAllReferenceStartIndexes devuelve posiciones donde empieza cada referencia.
Corta con indices: si una referencia empieza en 0 y la siguiente en 145, toma el texto de 0 a 145.
Guarda solo texto util: si despues de limpiar la referencia queda vacia, no la agrega a la lista.
Apa7ReferenceService.cs - APA + IEEE en el mismo corte 
private static List<int> FindAllReferenceStartIndexes(string text)
{
    var apaStarts = FindReferenceStartMatches(text)
        .Select(match => match.Index);
    var ieeeStarts = FindIeeeReferenceStartMatches(text)
        .Select(match => match.Index);

    return apaStarts
        .Concat(ieeeStarts)
        .Distinct()
        .Order()
        .ToList();
}
Dos busquedas: una encuentra inicios APA y otra encuentra inicios IEEE.
Concat: junta ambas listas de posiciones.
Distinct: quita posiciones repetidas.
Order: ordena las posiciones de menor a mayor para cortar el texto en secuencia.
Apa7ReferenceService.cs - clasificacion de formato 
private static string DetectCitationStyle(string reference)
{
    if (IsIeeeReference(reference)) return "Formato IEEE";
    if (IsVancouverReference(reference)) return "Formato Vancouver";
    if (IsMlaReference(reference)) return "Formato MLA";
    if (IsChicagoReference(reference)) return "Formato Chicago";
    if (LooksLikeApa7(reference)) return "APA 7";

    return "Otro estilo";
}
Orden de preguntas: primero descarta formatos reconocibles que no son APA.
Solo al final pregunta APA: si no fue IEEE, Vancouver, MLA o Chicago, revisa si parece APA 7.
Otro estilo: si no encaja en ningun patron, queda como estilo no reconocido.

Apa7ReferenceService.cs - conteo por anio 
public static SortedDictionary<int, int> CountYears(IEnumerable<string> references)
{
    var counts = new SortedDictionary<int, int>();

    foreach (var reference in references)
    {
        var match = Regex.Match(
            reference,
            @"\(\s*(?:\d{1,2}\s+de\s+[A-Za-zÁÉÍÓÚÜÑáéíóúüñ]+\s+de\s+)?(?<year>(?:19|20)\d{2})[a-z]?\s*\)",
            RegexOptions.CultureInvariant);

        if (!match.Success || !int.TryParse(match.Groups["year"].Value, out var year))
        {
            continue;
        }

        counts[year] = counts.TryGetValue(year, out var currentCount)
            ? currentCount + 1
            : 1;
    }

    return counts;
}
counts: es un diccionario. La llave es el año y el valor es cuantas veces aparece.
Regex.Match: busca el año dentro de parentesis, por ejemplo (2020).
TryParse: convierte el texto del año en numero. Si no puede, ignora esa referencia.
Suma: si el año ya existe, aumenta el contador; si no existe, lo crea con valor 1.
Apa7ReferenceService.cs - fuente sin romper dominios .com 
private static IEnumerable<string> SplitSentencePartsPreservingDomains(string text)
{
    var protectedText = Regex.Replace(
        text,
        @"\b[A-Za-z0-9-]+\.(?:com|org|net|edu|gov|mx|es|ar|cl|co|br)\b",
        match => match.Value.Replace('.', '\u001F'));

    return Regex.Split(protectedText, @"\.\s+")
        .Select(part => part.Replace('\u001F', '.'));
}
Problema: algunas fuentes tienen punto dentro del nombre, como PortalesMedicos.com.
Proteccion: antes de dividir por puntos, cambia temporalmente el punto del dominio por un caracter especial.
Resultado: evita que PortalesMedicos.com se parta y termine contado solo como com.

5. Explicacion del codigo como clase basica

paso a paso

Esta seccion traduce el codigo a lenguaje de clase. Piensa que cada metodo es una estacion: recibe datos, hace una tarea pequena y devuelve un resultado para la siguiente estacion.

1

FindReferences(text): encontrar la bibliografia

Recibe todo el texto del PDF. Primero busca la seccion de referencias. Si no la encuentra, devuelve una lista vacia. Si la encuentra, manda esa seccion a separar.

2

SplitReferences(section): partir el bloque en piezas

Toma el bloque de referencias y busca todos los lugares donde parece iniciar una nueva referencia. Luego corta desde un inicio hasta el siguiente. Eso convierte un texto largo en una lista de referencias.

3

FindAllReferenceStartIndexes(text): guardar posiciones

No corta todavia. Solo apunta posiciones dentro del texto. Por ejemplo: referencia 1 empieza en el caracter 0, referencia 2 empieza en el caracter 145, referencia 3 empieza en el 302.

4

DetectCitationStyle(reference): decidir el estilo

Ya con una referencia aislada, pregunta: "esto parece APA 7?, IEEE?, Vancouver?, MLA?, Chicago?". Si no es APA 7, no se analiza con reglas APA.

5

AnalyzeReference(...): revisar la referencia

Si la referencia parece APA 7, revisa reglas: autor correcto, año despues del autor, titulo, tipo de fuente, DOI/URL, editorial, volumen, paginas y otros detalles.

6

CountYears / CountTitles / CountFirstAuthors / CountPublishers: contar

Estos metodos solo reciben referencias que ya pasaron como APA 7 correctas. Por eso el conteo final no mezcla referencias con errores ni referencias de otros formatos.

Resumen para recordar: una funcion no debe hacer todo. Aqui cada funcion tiene una tarea: encontrar, separar, clasificar, validar o contar.

6. Regex clave

patrones tecnicos

Separacion de referencias

Inicio APA:
Apellido, I. (año).
Institucion. (año).

Inicio IEEE:
Apellido, I., ... (año).
Apellido, I.: ... (año).

Corte mixto:
detecta inicio APA + inicio IEEE
separar cada referencia por su propio inicio

contarAnios(referenciasCorrectas)
contarTitulos(referenciasCorrectas)
contarPrimerosAutores(referenciasCorrectas)
